# Module 2: Attacks, Concepts And Techniques

This module explores various types of cyber threats, attack methods, and system vulnerabilities. It highlights how attackers exploit both technical weaknesses and human behavior, while emphasizing the importance of proactive security practices.

---

## Section 2.1: Malware and Its Impact

Malware is malicious software designed to disrupt systems, steal sensitive data, or gain unauthorized access.

### Key Concepts

- Spyware — Tracks user activity and captures sensitive information  
- Adware — Displays unwanted advertisements  
- Ransomware — Encrypts files and demands payment  
- Rootkits — Provide hidden, persistent access to systems  
- Viruses — Attach to files and require user execution  
- Worms — Self-replicate and spread across networks  
- Trojan Horses — Disguise themselves as legitimate software  
- Backdoors — Bypass authentication mechanisms  
- Scareware — Uses fear tactics to trick users  

### Real-World Examples

- A user downloads a free application that secretly installs spyware to capture login credentials  
- A ransomware attack locks a company’s files until payment is made  

### My Practical Understanding

Malware often operates silently, making it difficult to detect until damage is done. Understanding different types helps in early identification and response.

### How I Apply This

- Avoid downloading files from untrusted sources  
- Use antivirus software and keep it updated  
- Monitor systems for unusual behavior

## Section 2.2: Cyber Attacks and Techniques

Cyber attacks use both technical exploitation and human manipulation to compromise systems.

### Key Concepts

- Social Engineering — Manipulating individuals to reveal confidential information  
- Pretexting — Creating false scenarios  
- Tailgating — Gaining unauthorized physical access  
- Quid Pro Quo — Offering incentives for information  
- Denial-of-Service (DoS) — Overwhelming systems to disrupt services  
- Distributed DoS (DDoS) — Coordinated attack using multiple infected devices (botnets)  
- On-Path Attacks — Intercepting communication between two parties  
- SEO Poisoning — Redirecting users to malicious websites  
- Password Attacks — Includes brute-force, dictionary, and password spraying  
- Advanced Persistent Threats (APTs) — Long-term, targeted, stealthy attacks  
- Botnets — Networks of compromised devices used to launch attacks  

### Real-World Examples

- Attackers sending phishing emails pretending to be a bank to steal login details  
- A website becoming unavailable due to a DDoS attack from thousands of infected devices  

### My Practical Understanding

Many attacks succeed by exploiting human trust rather than technical weaknesses. Awareness and cautious behavior are essential for prevention.

### How I Apply This

- Verify requests before sharing sensitive information  
- Use strong and unique passwords  
- Avoid clicking suspicious links or search results  

---

## Section 2.3: System Vulnerabilities

Vulnerabilities are weaknesses in hardware or software that attackers exploit to gain access or cause damage.

### Key Concepts

**Hardware Vulnerabilities**
- Rowhammer — Memory-based attack  
- Meltdown and Spectre — CPU side-channel attacks  

**Software Vulnerabilities**
- Buffer Overflow — Exceeding memory limits  
- Non-validated Input — Improper input handling  
- Race Conditions — Timing-related flaws  
- Weak Security Practices — Poor implementation  
- Access Control Issues — Improper permission management  

**Security Measures**
- Regular updates and patching  
- Penetration testing  
- Secure configuration and access control  

### Real-World Examples

- A system crash caused by buffer overflow exploitation  
- Unauthorized access due to failure to revoke permissions of former employees  

### My Practical Understanding

Most vulnerabilities are caused by poor design or misconfiguration. Regular updates and proper security practices significantly reduce risks.

### How I Apply This

- Keep systems and software updated  
- Follow secure configuration practices  
- Regularly review user access permissions  

---

## Section 2.4: Emerging Threats

New technologies introduce new security risks that attackers can exploit.

### Key Concepts

- Cryptocurrency — Digital currency secured by cryptography and blockchain  
- Cryptojacking — Unauthorized use of system resources to mine cryptocurrency  

### Real-World Examples

- A compromised website using visitors’ devices to mine cryptocurrency without consent  
- Sudden system slowdown caused by hidden cryptojacking scripts  

### My Practical Understanding

Emerging technologies create new attack surfaces. Even legitimate innovations can be abused if not properly secured.

### How I Apply This

- Monitor system performance for unusual activity  
- Avoid suspicious websites and browser extensions  
- Use security tools to detect unauthorized processes
