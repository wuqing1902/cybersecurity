# Module 3: Protecting Your Data And Privacy

This module focuses on practical cybersecurity measures for protecting devices, networks, and personal data. It emphasizes secure configurations, user responsibility, and real-world defensive strategies against modern threats.

---

## Section 3.1: Device and Network Security

Securing computing devices and networks is the first line of defense against cyber threats.

### Key Concepts

- Enable and maintain firewalls to block unauthorized access  
- Install antivirus and antispyware from trusted sources  
- Keep operating systems and browsers updated with security patches  
- Use strong passwords and encrypt sensitive data  
- Isolate IoT devices on separate networks  

**Wireless Security**
- Change default SSID and router passwords  
- Enable WPA2 encryption  
- Avoid relying solely on hidden SSID  

**Public Wi-Fi Risks**
- Avoid transmitting sensitive data on public networks  
- Disable file and media sharing  
- Use VPN for encrypted communication  
- Turn off Bluetooth when not in use  

**Password Security**
- Use unique passwords for each account  
- Use password managers to store credentials securely  
- Prefer passphrases over simple passwords  
- Follow recommended security standards (e.g., NIST guidelines)  

### Real-World Examples

- A hacker gaining access through a default router password  
- Data being intercepted on public Wi-Fi without encryption  

### My Practical Understanding

Basic security configurations are often overlooked, making devices easy targets. Strong passwords, updates, and network security significantly reduce risk.

### How I Apply This

- Regularly update devices and applications  
- Use password managers and strong passphrases  
- Avoid using public Wi-Fi without a VPN  

---

## Section 3.2: Data Protection and Encryption

Protecting data ensures confidentiality and prevents unauthorized access or loss.

### Key Concepts

- Encryption converts data into unreadable form using keys  
- Encryption protects data even if intercepted  
- Ransomware can misuse encryption to lock user data  
- Use tools like Encrypting File System (EFS) for file protection  

**Backups**
- Perform regular backups to external drives or cloud storage  
- Use cloud services to protect against physical damage or theft  

**Permanent Deletion**
- Deleted files can still be recovered from storage devices  
- Proper data wiping is required for complete removal  

### Real-World Examples

- Losing important files due to hardware failure without backups  
- Sensitive data recovered from improperly deleted files  

### My Practical Understanding

Data protection is not only about preventing access but also ensuring recovery. Backups and proper deletion methods are essential.

### How I Apply This

- Perform regular backups (local and cloud)  
- Encrypt sensitive files  
- Ensure proper deletion of confidential data  

---

## Section 3.3: Privacy, Terms of Service & User Control

Users must understand how their data is collected, used, and shared by online services.

### Key Concepts

- Terms of Service (ToS) define how user data is handled  
- Companies may gain rights to reuse or distribute user content  
- Default privacy settings are often set to public  
- Users should review security and privacy policies before use  

### Real-World Examples

- Personal content being reused by platforms due to ToS agreements  
- Public profiles exposing sensitive personal information  

### My Practical Understanding

Privacy is often compromised due to lack of awareness. Users must actively manage their data and understand platform policies.

### How I Apply This

- Review ToS and privacy policies before signing up  
- Adjust privacy settings to limit data exposure  
- Avoid sharing sensitive personal information online  

---

## Section 3.4: Authentication and Online Privacy

Strong authentication and careful online behavior are essential to protect digital identity.

### Key Concepts

- Two-Factor Authentication (2FA) adds an extra security layer  
- OAuth allows secure login via trusted third-party providers  
- Limit personal information shared on social media  

**Email and Browser Privacy**
- Emails can be intercepted during transmission  
- Browsing history can be accessed if devices are compromised  
- Use incognito/private browsing to reduce stored data  

### Real-World Examples

- Accounts compromised due to lack of 2FA  
- Personal data exposed through social media oversharing  

### My Practical Understanding

Authentication is a critical defense layer. Even strong passwords are insufficient without additional verification methods.

### How I Apply This

- Enable 2FA on all important accounts  
- Use trusted authentication methods  
- Limit personal information shared online  

---

## Section 3.5: Practical Security Scenarios

Applying cybersecurity knowledge in real-world situations is essential for effective protection.

### Key Concepts

- Avoid posting sensitive information (e.g., location, travel plans)  
- Always use VPN on public networks  
- Do not install software from untrusted sources  
- Be cautious of scareware and fake alerts  
- Identify phishing attempts through suspicious emails and URLs  

### Real-World Examples

- A burglar targeting a home after vacation posts on social media  
- Malware infection from downloading untrusted "free" software  

### My Practical Understanding

Real-world attacks often rely on simple mistakes. Awareness and cautious behavior significantly reduce exposure to threats.

### How I Apply This

- Avoid oversharing on social media  
- Verify software sources before installation  
- Stay alert to phishing and suspicious messages
